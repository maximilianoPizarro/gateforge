GateForge

AI-powered migration platform for transitioning from Red Hat 3scale API Management to Red Hat Connectivity Link (Kuadrant) on OpenShift.

Start here

Installation

Install GateForge using Helm on OpenShift or run locally with Podman Compose.

# Add the Helm repo
helm repo add gateforge \
  https://maximilianopizarro.github.io/gateforge/

# Install
helm install gateforge gateforge/gateforge

Local Development

Run the full stack locally with Podman Compose for development and testing.

# Clone the repo
git clone https://github.com/maximilianoPizarro/gateforge.git

# Build and run
podman-compose up -d --build

Configuration

All parameters are configurable via Helm values: AI model, 3scale credentials, gateway strategy, and more.

helm install gateforge gateforge/gateforge \
  --set ai.apiKey=YOUR_KEY \
  --set threescale.adminApi.url=URL \
  --set connectivityLink.gatewayStrategy=shared

GateForge includes an AI-powered chat assistant that helps you analyze, plan, and execute your 3scale to Connectivity Link migration. Here are some example prompts to get started:

Try these prompts:

🛡️ Policy Generation

Create a RateLimitPolicy equivalent to my 3scale Application Plan that limits 100 requests per minute per user

🌐 Route & Gateway Generation

Try these prompts:

Convert my 3scale mapping rules into Kubernetes Gateway API HTTPRoute resources with proper path and method matching

How do I configure TLSPolicy for my migrated APIs?

Gateway Strategy Comparison

StrategyDescriptionBest For
sharedAll APIs share a single Istio Gateway with multiple HTTPRoutesSmall to medium clusters, simpler management
dualTwo Gateways (internal/external) with APIs split by exposure levelOrganizations requiring network segmentation
dedicatedOne Gateway per 3scale Product, full isolation per APILarge enterprises, strict isolation requirements
